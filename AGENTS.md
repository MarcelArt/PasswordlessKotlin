# AGENTS.md

Guidance for AI agents working in this repository.

## Scope of allowed edits

- You may ONLY modify files under:
  `shared/src/commonMain/kotlin/art/bangmarcel/passwordlesskotlin/screens/`
- Inside that directory, work is limited to composables: screens, components, and widgets.
- Do NOT edit ViewModels (`viewmodels/`), repositories, models, platform code, build scripts, or anything outside the directory above. If a change requires touching a ViewModel, stop and ask the user — they will update it themselves.
- More directories may be allowlisted later. Do not assume other paths are fair game.

## Things you must NOT do autonomously

- Do not run any Gradle command (`./gradlew ...`, `gradle ...`). If you need a build, test, or codegen run, ask the user and they will run it.
- Do not install, add, upgrade, or remove dependencies (no edits to `gradle/libs.versions.toml`, `*.build.gradle.kts`, or `settings.gradle.kts`). You may suggest changes; the user will apply them.
- Do not edit ViewModels. Propose the change and let the user do it.

## Styling rules (enforced)

- Never use hardcoded values for color, typography, shape, etc. Examples of what NOT to do: `Color(0xFF080808)`, hex strings, fixed font sizes bypassing the theme.
- Always source visuals from the Material3 theme tokens:
  - Colors → `MaterialTheme.colorScheme.*` (e.g. `surface`, `onSurface`, `primary`, `onSurfaceVariant`, `error`)
  - Typography → `MaterialTheme.typography.*`
  - Shapes → `MaterialTheme.shapes.*`
- This applies to every composable, component, and widget you write or edit.

## Documentation lookup

- Always use the Context7 MCP skill/tool to fetch current API docs for any library touched (Compose Multiplatform, Voyager, Ktor, Material3, kotlinx.serialization, etc.).
- Do not rely on memory for API signatures, names, or version-specific behavior. If you cannot verify it via Context7, do not write it.

## Project shape (so you don't have to guess)

- Kotlin Multiplatform + Compose Multiplatform. Targets: Android, iOS, Desktop (JVM).
- Gradle modules: `:shared` (common UI + logic), `:androidApp`, `:desktopApp`, plus the Xcode-based `iosApp`.
- Shared code lives in `shared/src/commonMain/kotlin/art/bangmarcel/passwordlesskotlin/` with subpackages:
  - `screens/` — Voyager `Screen` composables (your editable surface)
  - `viewmodels/` — Voyager `ScreenModel` classes (do NOT edit; ask first)
  - `repositories/`, `models/`, `configs/`, `utils/` — supporting code (do NOT edit)
  - Platform-specific code under `androidMain`, `iosMain`, `jvmMain` (do NOT edit)
- App entry: `App.kt` builds the Ktor `HttpClient` + `AuthRepo`, wraps content in `MaterialTheme { Navigator(BeginRegisterScreen(...)) }`.
- Navigation + ViewModels: **Voyager** (`cafe.adriel.voyager`). Screens implement `Screen`; models extend `ScreenModel` and are obtained via `rememberScreenModel { ... }`.
- Networking: **Ktor** client with CIO engine + `ContentNegotiation`/`kotlinx.json`. Serialization via `kotlinx.serialization`.
- UI: **Material 3** (`androidx.compose.material3`). Compose Multiplatform `1.11.1`, Kotlin `2.4.0`.
- Build config (`apiBaseUrl`, etc.) is generated by the **buildkonfig** plugin into `art.bangmarcel.passwordlesskotlin.BuildKonfig`. Override per environment via `local.properties` (gitignored), not via source edits.

## Running the app (user runs these — do not run yourself)

- Android: `./gradlew :androidApp:assembleDebug`
- Desktop standard: `./gradlew :desktopApp:run`
- Desktop hot reload: `./gradlew :desktopApp:hotRun --auto`
- iOS: open `iosApp/` in Xcode and run.

## Tests

- No meaningful test suite exists yet. `commonTest` has only the default stub. Do not invent test commands.
